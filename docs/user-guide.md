# User guide

## What Faraid does

Faraid computes the Islamic inheritance shares for a given set of heirs
under any of the four classical Sunni schools (Hanafi, Maliki, Shafi'i,
Hanbali). The engine uses exact rational arithmetic: every share is
returned as an irreducible fraction such as `1/4` or `2/3`, so there is
no floating-point rounding.

## The determinism guarantee

The legal core (`internal/core`) is a pure function: given the same
input it always produces the same output. It performs no I/O, makes no
network calls, and does not read from the environment. Every rule is
encoded as a Go function covered by a test, and every test case derives
from a classical reference problem.

The derivation log returned with every result lists every step taken,
which school rule was applied, and the classical reference for that rule.

## How to use the calculator

1. Select whether the deceased was **male** or **female**.
2. Select the **school of law** (madhab).
3. Optionally enter the **estate value** (total assets minus liabilities)
   and the deductions (funeral, debts, bequests).
4. Enter the **heirs** using the +/- counters. Each heir group shows
   which heirs may block others.
5. Click **Calculate Shares**.

The result table shows each heir's fraction and, if you entered an
estate value, their monetary amount.

The **Step-by-step Derivation** panel shows every rule that was applied:
estate deductions, blocking (hajb), fixed shares (furud), residuary
(asaba), and any awl or radd adjustment.

## Comparing schools

The **Compare Schools** view runs the same case through all four schools
simultaneously and highlights heirs whose shares diverge across schools.

## Saving and sharing

After calculating, click **Copy share link** to save the case and copy a
URL to the clipboard. Opening that URL restores the exact inputs so you
can share a case with a colleague or return to it later.

Click **Download PDF** to download a printable summary of the result.

## The trial LLM explanation (experimental)

When the administrator has enabled the trial tier
(`PUBLIC_LLM_ENABLED=true`), a button labelled **Explain in plain
language (experimental)** appears after a result. Clicking it asks an
AI model to rephrase the derivation in natural language.

**Important limitations:**
- The AI explanation is generated by a large language model and is
  **not authoritative**. It does not constitute a legal ruling (fatwa).
- The explanation is checked against the engine's fractions. If the AI
  contradicts the computed numbers, a warning is displayed.
- The engine result is always the ground truth. When in doubt, consult
  the derivation log and a qualified Islamic scholar.

## The natural-language parser (experimental)

Also behind the trial flag: a text box where you can describe a case in
natural language (Arabic or English), for example:

> "A man died leaving his wife, two sons, and a daughter."

The parser uses an AI model to extract the case, then shows it for
**human review before any calculation**. You must click **Apply** to
load the parsed heirs into the form. Always verify the extracted data
against the original description.
